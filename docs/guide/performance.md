# 性能优化

针对声明式应用的常见优化策略：

## 组件与事件
- 保持业务函数纯净，避免在函数中频繁操作 DOM
- 将长耗时任务放入后端或使用队列（queue）减少前端阻塞
- 输出组件设置为 `interactive: false` 可避免不必要的交互事件

## 布局与渲染
- 使用 Row/Column 组合，减少嵌套层级
- 避免一次性渲染过多组件，可按需显示/隐藏
- 复用组件实例，减少销毁/创建开销

## 状态与持久化
- 合理选择持久化方式：local 与 session 适合用户偏好，url 适合分享状态
- 订阅仅在需要的地方使用，及时取消订阅

## 资源与样式
- 按需加载图片资源，降低首屏占用
- 通过主题变量进行样式统一管理，减少内联样式与复杂选择器

## 调试建议
- 使用浏览器性能面板检查长任务与内存占用
- 分阶段测试：先验证组件与事件，再接入状态与服务端
